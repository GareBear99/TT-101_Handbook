# Timeproof Continuity Protocol (TPCP)

A protocol designed to be valid under **any** of these regimes:
- fixed/self-consistent time (block universe),
- branching time,
- no time travel.

## 0) Ethos
- Minimal impact
- No coercion, no violence
- Consent-forward
- Preserve ordinary history as much as possible
- Optimize personal continuity, not civilization-level rewrites

## 1) Define the Pivot Window
Choose a 3–12 month window where:
- You can take action quickly
- Your environment is stable enough
- A small nudge changes long-term trajectory

Use: `templates/Pivot-Window-Worksheet.md`

## 2) Minimal-Impact Message Budget (MIMB)
The future contact may deliver only:
- 1 identity proof
- 1 direction
- 1 warning
- 1 tool (checklist / method)

No tech dumps. No markets. No geopolitics.

Use: `templates/Minimal-Impact-Message.md`

## 3) Authentication Handshake
Use a 3-part handshake:
- Phrase seed (private, never spoken)
- Artifact proof (physical + verifiable structure)
- Behavioral proof (questions only a real continuity agent can answer)

Use: `templates/Handshake-Protocol.md`

## 4) Distributed Archive Base
Build redundant storage across:
- physical print + sealed envelopes,
- offline encrypted drives,
- cloud mirrors under different providers,
- optional public “decoy layer” + private “core layer.”

Use: `protocols/Archive-Base.md`

## 5) Attractor Building
Create real-world “trajectory attractors”:
- skill compounding
- project shipping cadence
- financial runway
- health routines
- trusted collaborators

These reduce the chance that a single shock derails your life.

## 6) Delivery Rules (if contact occurs)
- private contact only
- no public demos
- no broad societal interventions
- log everything (append-only)
- stop if authentication fails

## 7) Versioning
- TT-101 v1.0 is conceptual.
- Any future updates should be appended, not overwritten.
