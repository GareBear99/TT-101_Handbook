# TT-660 — Self-Replication & the Fab Stack


This is not “instant grey goo.”
It is a staged pathway from **household objects** → **workshop** → **factory** → **outpost**.

## Stage 0 — Household Automation (Today)
- commodity 3D printers
- simple CNC/laser cutters
- basic robotics (arms, gantries, sliders)
- open-source control software

## Stage 1 — Workshop Replication
Objective: replicate the *tools that replicate tools*.
- print brackets, fixtures, housings
- machine rails/shafts
- salvage motors, bearings, fasteners
- build tool changers and calibration jigs

## Stage 2 — Materials Loop
- filament extrusion from plastic waste
- metal casting + machining for durable parts
- recycling and QA protocols

## Stage 3 — Compute Guidance
Even if fabrication is physical, **the plan is informational**:
- external scripts can supply build order
- verification tests and calibration routines
- deterministic “recipes” for assembly

## Reality Check (Electronics Bottleneck)
You cannot practically bootstrap modern semiconductors from scratch in a small outpost.
So the replication plan must include:
- chip stockpiles
- board rework capability
- modular compute with cold spares
- graceful degradation modes (TT-304)
