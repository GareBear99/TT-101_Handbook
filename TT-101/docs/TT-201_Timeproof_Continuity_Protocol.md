Core Components:

1) Pivot Window
- Defined time band where intervention has maximal leverage.
- Must be condition-based, not vague.
See: [TT-202 Pivot Window Model](TT-202_Pivot_Window_Model.md)

2) Minimal-Impact Message Budget
- 1 identity proof
- 1 direction
- 1 warning
- 1 tool
No civilization-scale disruption.
See: [TT-203 Minimal Message Architecture](TT-203_Minimal_Message_Architecture.md)

3) 3-Part Handshake
- Private Phrase Seed
- Artifact Proof
- Behavioral Verification
See: [TT-204 Handshake Specification](TT-204_Handshake_Spec.md)

4) Distributed Archive Base
- Redundant locations
- Hash-chained manifest / integrity checks
- Human-readable fallback formats
See: [TT-205 Distributed Archive Specification](TT-205_Distributed_Archive_Spec.md)

5) Temporal Beacon (optional)
- Publicly verifiable pivot marker (“listen here”).
- Binds pivot windows to integrity artifacts.
- Adds a **Listener Contract** to constrain responses.
See: [TT-206 Temporal Beacon Protocol](TT-206_Temporal_Beacon_Protocol.md)

6) Attractor Engineering
- Skill stacking
- Resilience building
- Capital preservation
- Health and cognitive continuity

Purpose:
Guarantee usefulness whether time travel exists or not.
